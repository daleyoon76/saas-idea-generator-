# 2026년 에이전틱 SaaS 초기 아이템 발굴 기준

## 거시적 시장 환경 (2026년)

- 글로벌 SaaS 시장에서 "SaaSpocalypse" 발생 — 수퍼 에이전트 등장으로 소프트웨어 시가총액 약 2조 달러 증발
- **좌석 압축(Seat Compression)**: AI 에이전트 1개가 수십 명분 라이선스를 대체
- 가트너 전망: 2030년까지 기존 SaaS 도구 35%가 AI 에이전트로 대체 또는 흡수
- **핵심 전환**: "사람이 조작하는 도구" → "스스로 일하는 디지털 노동력"

## 5대 아이템 발굴 기준

### 기준 1: 수퍼 에이전트 시대의 수요 형태 변화 (핵심)

- 소프트웨어 가치의 중심이 **"도구 접근권"에서 "결과물"**로 이동
- 수요 형태: 동기식(화면 조작) → **비동기식(백그라운드 위임, 결과 확인)**
- **핵심 질문**: "사용자가 잠든 사이 이 에이전트가 어떤 복잡한 업무를 끝내놓을 수 있는가?"

### 기준 2: 마이크로 니치 기반 버티컬 워크플로우 타게팅

- 빅테크(OpenAI·Google·MS)가 수평적 AI 시장 장악 → **버티컬 특화만이 살 길**
- 버티컬 에이전트 시장은 기존 SaaS 시장 대비 최대 10배 규모 가능성(Y Combinator)
- 사용자 행동 데이터 → 에이전트 성능 향상 → 전환 비용 상승 (강력한 해자 형성)

### 기준 3: 결과 기반(Outcome-based) 수익화 모델 적용 가능성

- 좌석 기반 과금 한계 봉착 → **사용량·성과 기반 과금으로 전환** (SaaS 기업 85%+ 채택)
- 과금 기준 예시: API 호출 수, 성사된 미팅 건수, 심사 정확도 향상률
- 하이브리드 모델 (정액 기본료 + 성과 변동 수수료)이 가장 현실적

### 기준 4: 바이브 코딩 기술 타당성 극대화 & 보안 한계 회피

- AI 생성 코드의 약 **24.7%가 잠재적 보안 결함** 포함
- **피해야 할 영역**: 금융 원장 동기화, 고주파 트레이딩, 복잡한 멀티스레드 처리
- **적합한 영역**: 결함 발생 시 빠른 롤백 가능, 시각적 요소 중심, 가벼운 워크플로우 자동화

### 기준 5: 에이전틱 UX & 공유된 자율성(Shared Autonomy) 설계

- 완전 자동화보다 **"AI 초안 + 인간 최종 승인"** 워크플로우가 채택률 높음
- 반드시 포함해야 할 **자율성 제어 모드**: 관전 / 지원 / 자율
- 감사 로그(Audit log) UI 제공 필수

## 아이템 선정 체크리스트

1. 사용자가 화면 없이 결과만 받아도 되는 비동기 워크플로우인가?
2. 빅테크가 쉽게 침범하기 어려운 마이크로 니치 버티컬인가?
3. 성과를 숫자로 증명하고 과금 근거로 삼을 수 있는가?
4. 에이전트 실수 시 롤백 가능하고 보안 리스크가 낮은 영역인가?
5. 인간이 통제권을 느낄 수 있는 Shared Autonomy UX를 설계할 수 있는가?
